1. 📌 /*What is Salesforce Platform?*/

The **Salesforce Platform** is a powerful cloud-based platform used to build business applications, automate workflows, manage customer relationships, and develop secure enterprise solutions.

It supports:

✅ CRM (Customer Relationship Management)  
✅ No-Code & Low-Code Development  
✅ App Development  
✅ Automation  
✅ AI Integration  
✅ APIs & Integrations  
✅ Secure Cloud Infrastructure  

---

2. 🏗️ /*Core Salesforce Concepts*/

## 📱 1. App

An **App** in Salesforce is a collection of tools, tabs, objects, and features designed for a specific business purpose.

### 🔹 Examples
- Sales App
- Service App
- Dreamhouse App

### 💡 Simple Understanding
An App is like a **workspace** where users perform their daily tasks.

---

## 📦 2. Object

An **Object** is similar to a database table that stores data in Salesforce.

### 🔹 Types of Objects

| Type | Examples |
|---|---|
| Standard Objects | Account, Contact, Lead |
| Custom Objects | Property, Broker |

### 💡 Simple Understanding
Objects help organize and store information.

Example:
- Contact Object → Stores customer details
- Property Object → Stores property details

---

## 🗂️ 3. Tab

A **Tab** is used to access objects, records, dashboards, and applications from the Salesforce interface.

### 🔹 Examples
- Accounts Tab
- Contacts Tab
- Properties Tab

### 💡 Simple Understanding
Tabs help users quickly navigate through Salesforce.

---

3. ⚡ /*Configuration vs Coding*/

| 🔧 Configuration | 💻 Coding |
|---|---|
| No-code / Low-code approach | Programmatic development |
| Uses drag-and-drop tools | Uses programming languages |
| Faster implementation | More advanced customization |
| Mainly done by Admins | Mainly done by Developers |
| Uses Flow Builder, App Builder | Uses Apex, LWC, APIs |
| Easy for beginners | Requires coding knowledge |
